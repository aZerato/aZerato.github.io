Windows Server 2003 : HTTPS 502

Vous effectuez des requêtes HTTPS mais vous obtenez directement un code d'erreur 502 ?

Il était nécessaire de requêter un WebService SSO côté server (pour limiter la plage IP à autoriser au niveau du server qui héberge le WS) par code C#. Seulement le code fonctionnant très bien sur une machine avec Windows 8, retournait une erreur "System.IO.IOException: Received an unexpected EOF or 0 bytes from the transport stream" sur le server Windows 2003.

Il fut nécessaire de faire une mise à jour permettant de revoir le système d'encryption pour le requêtage HTTPS.

Windows Server 2003 : HTTPS 502

An HTTPS request returns 502 error.

For ip limitation, we need to request a WS without ajax but directly with c#. The request code works successfully on windows 8, but isn't on the Windows Server 2003. I've got this "System.IO.IOException: Received an unexpected EOF or 0 bytes from the transport stream".

A Microsoft update patch HTTPS encryption.
